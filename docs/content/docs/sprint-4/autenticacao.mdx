## Visão Geral

Na Sprint 4, consolidamos a estratégia de segurança do ecossistema. Dado que o projeto envolve APIs de interface, processamento assíncrono (Kafka) e serviços gerenciados no Google Cloud Platform (GCP), a segurança foi segmentada em duas abordagens complementares.

1.  **Autenticação de Aplicação (JWT)**: Focada no usuário final e controle de acesso baseado em papéis (RBAC).
2.  **Autenticação de Infraestrutura (GCP IAM)**: Focada na comunicação segura entre serviços e recursos de nuvem.

---

## 1. Auth-middleware (JWT)

O serviço `Auth-middleware`, desenvolvido em Rust com Axum, centraliza a emissão e validação de tokens. Implementamos três níveis de acesso principais:

| Papel (Role) | Descrição |
| :--- | :--- |
| **User** | Acesso básico para visualização de dashboards e consultas simples. |
| **Administrator** | Controle total do sistema, gestão de usuários e configurações críticas. |
| **Eletromidia-Employee** | Operadores técnicos com acesso a monitoramento e gestão de inventário. |

### Endpoints de Autenticação

*   `POST /authenticate`: Recebe credenciais e retorna um token JWT assinado.
*   `POST /authorize`: Valida se um token é legítimo e possui o papel (`Role`) necessário.
*   `GET /me`: Permite que a interface recupere os dados do usuário logado a partir do token.

## 2. Aplicabilidade e Arquitetura

A segurança é aplicada de forma distinta dependendo da natureza do serviço:

### Serviços com Proteção JWT
Estes serviços expõem endpoints HTTP para clientes ou administradores e utilizam o `AuthToken` extractor:

*   **`bff-storage`**: Protege o upload de arquivos para o Cloud Storage.
*   **`agent`**: Garante que as consultas de IA (LLM) sejam feitas apenas por usuários autorizados.
*   **`rust-producer`**: Restringe o disparo de testes de carga e ingestão manual de CSVs a administradores.

### Serviços com Proteção GCP IAM
Serviços de background (workers) que não possuem interface pública utilizam identidades de serviço (`Service Accounts`) nativas do GCP:

*   **`bucket-consumer`**: Autorizado via IAM para escrever objetos no Cloud Storage.
*   **`bigquery-writer`**: Autorizado para inserir linhas no Data Warehouse (BigQuery).
*   **`populate-bigquery`**: Job de ETL que utiliza permissões de leitura/escrita no Storage e BigQuery.
