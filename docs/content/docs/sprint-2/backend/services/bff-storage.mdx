## O que é

O `bff-storage` é o serviço de **Backend for Frontend (BFF)** responsável por expor, via HTTP, os arquivos armazenados no bucket de histórico (`kafka-backup-eletromidia`).

Na prática, ele oferece uma API simples para:

- listar objetos do bucket inteiro;
- listar objetos por pasta/prefixo;
- baixar um arquivo específico por ID.

Ele é o ponto de acesso da interface para dados históricos já persistidos no Object Storage.

---

## Onde se encaixa na arquitetura

```text
Kafka Topic: geodata
      │
      ▼
bucket-consumer
      │
      ▼
Google Cloud Storage (bucket kafka-backup-eletromidia)
      │
      ▼
bff-storage
      │
      ▼
UI / cliente HTTP
```

O fluxo de arquivos é complementar ao [BigQuery Writer](/docs/sprint-2/backend/services/bigquery-writer): enquanto o writer estrutura dados para analytics no DW, o BFF Storage entrega os artefatos brutos/NDJSON diretamente do bucket.

## Endpoints HTTP

Todos os endpoints aceitam apenas método `GET`.

| Endpoint | Query params | Sucesso | Erros comuns | Uso |
| --- | --- | --- | --- | --- |
| `/bucket/items` | - | `200` | `500`, `405` | Lista todos os objetos do bucket |
| `/bucket/items/by-folder` | `folder` (obrigatório) | `200` | `400`, `500`, `405` | Lista objetos por prefixo |
| `/bucket/items/file` | `id` (obrigatório) | `200` | `400`, `404`, `500`, `405` | Baixa um arquivo específico |

### Exemplo de request/response: listagem por pasta

Requisição:

```bash
curl -X GET "http://<MINIKUBE_IP>:<NODE_PORT>/bucket/items/by-folder?folder=kafka-backup/topics/geodata/year=2026/month=02/day=27"
```

Resposta (`200 OK`):

```json
{
  "bucket": "kafka-backup-eletromidia",
  "folder": "kafka-backup/topics/geodata/year=2026/month=02/day=27/",
  "count": 1,
  "items": [
    {
      "id": "kafka-backup/topics/geodata/year=2026/month=02/day=27/2-41-41.json",
      "size": 538,
      "content_type": "application/json",
      "updated_at": "2026-02-27T12:29:32Z"
    }
  ]
}
```

### Exemplo de request/response: download de arquivo

Requisição:

```bash
curl -X GET "http://<MINIKUBE_IP>:<NODE_PORT>/bucket/items/file?id=kafka-backup/topics/geodata/year=2026/month=02/day=27/2-41-41.json" -O
```

Resposta (`200 OK`):

- Body em streaming com o conteúdo do arquivo solicitado.
- Header `Content-Type: application/json`.
- Header `Content-Disposition: attachment; filename="2-41-41.json"`.
- Header `Content-Length: <tamanho_em_bytes>` (quando disponível).

## Regras de comportamento

### Normalização de pasta (`folder`)

No endpoint `/bucket/items/by-folder`, o serviço:

- remove `/` no início;
- garante `/` no final;
- rejeita vazio com `400`.

Exemplo:

```text
folder=/kafka-backup/topics/geodata/year=2026/month=02/day=27
↓
kafka-backup/topics/geodata/year=2026/month=02/day=27/
```

### Normalização de arquivo (`id`)

No endpoint `/bucket/items/file`, o serviço:

- remove espaços e `/` inicial;
- retorna `404` quando o objeto não existe;
- envia resposta em streaming (`io.Copy`) com:
  - `Content-Type` do objeto (ou `application/octet-stream`);
  - `Content-Disposition: attachment; filename="<nome-base>"`;
  - `Content-Length` quando disponível.

### Filtragem de pseudo-diretórios

Objetos terminados em `/` são ignorados nas listagens para não exibir marcadores de pasta como arquivos reais.

### Timeouts e robustez

- `ReadHeaderTimeout`: `5s` no servidor HTTP.
- `context.WithTimeout`: `30s` para operações no Cloud Storage por requisição.

---

## Decisões de design

### API com `net/http`

O serviço usa `http.NewServeMux` e handlers diretos para reduzir dependências e manter o binário simples para execução em contêiner.

### Download por streaming

Arquivos são enviados do GCS para o cliente sem carregar tudo em memória, o que evita picos de RAM com objetos maiores.

### Autenticação por chave de Service Account

O cliente GCS é inicializado com `option.WithCredentialsFile`, usando caminho vindo da variável `CS_SA_CREDENTIALS` e montado por `Secret` no pod.

## Kubernetes

### `configmap.yaml`

Define `BUCKET_NAME` e caminho de credencial (`/secrets/cs-api-key.json`).

### `deployment.yaml`

- 1 réplica;
- imagem `bff-storage:latest` com `imagePullPolicy: Never` (fluxo local com minikube);
- requests/limits de CPU e memória;
- montagem do secret `gcs-sa-key` em `/secrets/cs-api-key.json`.

### `np-service.yaml`

- serviço `NodePort`;
- porta de serviço `5678` direcionando para `8080` no container.
