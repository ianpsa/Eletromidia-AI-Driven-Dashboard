## Visão geral

O frontend em `src/frontend` é uma SPA em React + TypeScript, construída com Vite e CSS puro. A aplicação concentra quatro áreas:

- `Login`, que libera o acesso inicial à interface
- `Dashboard`, com embed do Looker Studio
- `Cloud`, com navegação e download de arquivos do bucket
- `Agente IA`, com envio de prompts para análise

O controle de navegação é local, feito em memória no `App.tsx`. Não há rotas por URL nem persistência de sessão após recarregar a página.

## Fluxo da aplicação

```text
Login
  │
  ▼
Dashboard
  ├─ Cloud
  └─ Agente IA
```

Após o login, o estado inicial da aplicação aponta para `Dashboard`. A troca entre telas acontece pelo menu lateral.

## Onde o BFF entra

O escopo mais próximo do BFF Storage está na tela `Cloud`.

```text
Navegador
  ├─ Login                → estado local
  ├─ Dashboard            → iframe do Looker Studio
  ├─ Cloud                → BFF Storage
  └─ Agente IA            → endpoint /analyze
```

Na prática, o frontend espera os seguintes contratos:

| Área        | Origem                                | Uso no frontend                                                      |
| ----------- | ------------------------------------- | -------------------------------------------------------------------- |
| `Cloud`     | `GET /bucket/items/by-folder?folder=` | Carrega `bucket`, `folders` e `items` do nível atual                 |
| `Cloud`     | `GET /bucket/items/file?id=`          | Faz o download do arquivo e resolve o nome via `Content-Disposition` |
| `Dashboard` | Looker Studio embed                   | Exibe o dashboard analítico dentro de um `iframe`                    |
| `Agente IA` | `POST /analyze`                       | Envia o prompt e recebe texto com ranking                            |

## Tela Cloud

![Tela Cloud do frontend](/sprint-3/frontend/cloud.png)

A tela `Cloud` é a interface diretamente integrada ao BFF Storage. Ela combina navegação hierárquica, busca local e download no mesmo fluxo.

- O breadcrumb parte de `Raiz` e deriva do `currentFolder`
- A busca filtra apenas os itens já carregados no nível atual
- A ordenação existe para `nome` e `tamanho`, mantendo pastas antes de arquivos
- Os cards de resumo mostram contagem de pastas, arquivos e tamanho total do nível atual
- O download usa `fetch`, cria um `blob` no navegador e aplica o nome final do arquivo

## Configuração

| Variável                | Default                 | Finalidade                             |
| ----------------------- | ----------------------- | -------------------------------------- |
| `VITE_BASE_URL`         | `/api`                  | Prefixo usado nas chamadas do frontend |
| `VITE_DEV_PROXY_TARGET` | `http://localhost:8080` | Destino do proxy no ambiente local     |
